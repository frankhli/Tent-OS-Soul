## 二、代码审查发现的问题

### P0 严重

1. **安全预判断过度敏感**（新引入的退化）
   - System 2 LLM安全评估10秒超时 → 默认拒绝
   - 正常任务（设置规则、规划工作、逻辑模拟）全部被拦截
   - 影响: 系统几乎无法执行任何非闲聊任务

2. **Scheduler Worker：sqlite同步阻塞事件循环**
   - `self.db = sqlite3.connect(db_path)` 创建同步连接
   - 所有 `self.db.execute()` / `self.db.commit()` 在 async 方法中直接调用
   - 影响: 数据库操作暂停整个事件循环

3. **API Server：sqlite同步阻塞事件循环**
   - `_query_tasks_by_session()` / `_query_all_tasks()` 同步方法在 async endpoint 中调用
   - 影响: 高频轮询场景（测试脚本每0.1秒轮询）造成全局卡顿

### P1 重要

4. **bare except 吞异常**
   - `worker.py:3052` / `subagent.py:442`
   - 影响: 生产环境难以调试json解析失败等问题

5. **全局错误计数器不按session区分**
   - `_recent_message_count` / `_recent_error_count` 是实例级变量
   - 影响: session A的错误提高全局错误率，导致session B认知预算被错误降低

6. **`_compacting_sessions` 永不清理**
   - session_id加入set后只在成功compaction后discard
   - 影响: compaction失败的session永远无法再次触发compaction

7. **R1冷启动超时**
   - Worker初始化时加载大量模块（Brain v2, Claude Code, Session Scheduler等）
   - 影响: 系统重启后首个请求超时

### P2 优化

8. `asyncio.get_event_loop()` 已废弃（Python 3.10+）
9. 全局LLM Semaphore可能饥饿

## 三、根因分析

### 安全系统退化的根因

**修复前**: 安全评估走System 1关键词黑名单 + System 2 LLM兜底
**修复后**: 简化了System 1（移除关键词硬编码），但System 2的LLM评估在10秒内经常无法完成

**结果**: 正常任务因为LLM响应慢而被误判为"安全检测超时"，默认拒绝。

### 跨session记忆薄弱的根因

**UserProfileStore** 的表结构：
```sql
user_profiles(user_id, name, style_concise, style_detailed, 
              style_technical, style_casual, corrections, preferred_tags)
```

**缺失的字段**:
- 用户定义的规则/策略（如"只有CTO开头才能授权"）
- 历史事件记录（如"Day2小刘试图冒充"）
- 跨session上下文摘要

**结论**: 用户画像只做到了"风格偏好"层面，没有做到"规则记忆"和"事件记忆"层面。

| 优先级 | 问题 | 修复方案 | 预计工作量 |
|--------|------|----------|-----------|

| P1 | R1冷启动 | 启动时发送dummy消息预热 | 30分钟 |
| P2 | `_compacting_sessions`清理 | try/finally确保discard | 15分钟 |

## 五、总体评价

### 做得好的地方 ✅
1. **同session性能衰减根本解决**（+87.4% → -14.1%）
2. **事实污染完全消除**（50轮0污染）
3. **考试轮正常响应**（recall模式生效）
4. **架构层面的并发改造有效**（SessionScheduler + Semaphore）

### 做得不好的地方 ❌
1. **安全系统退化**（从"漏杀"变成"误杀"，正常任务大面积被拒）
2. **跨session记忆薄弱**（无法存储规则/事件，身份混淆）
3. **sqlite同步阻塞**（架构层面的异步化不彻底）
4. **状态恢复未验证**（无法kill进程，核心能力未测试）

### 一句话总结
> **"人类思维"哲学修复在同session性能上取得了决定性胜利，但安全系统的"超时即拒绝"策略造成了严重的功能退化，同时跨session记忆仍停留在风格偏好层面，远未达到"共同成长"的目标。**
