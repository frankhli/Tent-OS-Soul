# Tent OS 视觉系统演进路线图

## 当前状态
- ✅ 面部情绪检测（FaceLandmarker 52 blendshapes）
- ✅ 情绪分析管道（竞争评分系统）
- ✅ 手动图片上传 → VLM 分析
- ❌ 摄像头画面从未离开浏览器（后端看不到图像）
- ❌ 没有持续环境感知

## Phase 1：定时场景快照（今晚就能跑）
**目标**：让后端"看见"房间，每 5 秒自动截一张图分析

### 技术方案
```
摄像头 → Canvas 截图(320×240) → JPEG 压缩(quality 0.6)
         → WebSocket binary/base64 → 后端 VLM
         → VisualMemoryService 自动存储
```

### 前端改动
- `useVision.ts` 新增 `captureScene()`：canvas 截视频帧
- 每 5000ms 自动截图，通过 `ws.send()` 发送 `vision.scene_frame`
- 截图前检测画面变化（与上一帧做像素差异），无变化不发

### 后端改动
- `server.py` 新增 WS 消息 `vision.scene_frame`
- 接收 JPEG → VLM (`vision_llm`) → 结构化场景描述
- 自动调用 `VisualMemoryService.store_memory()`

### 预期效果
- AI 开始"看见"你在什么环境（办公室/卧室/会议室）
- 自动记录"场景时间线"（10:00 在厨房，10:05 在客厅）
- 可以问"我昨天把水杯放哪了"

### 工作量：~2 小时

## Phase 2：前端物体检测（MediaPipe ObjectDetector）
**目标**：实时识别"这是什么"，不需要后端参与

### 技术方案
```
摄像头 → MediaPipe ObjectDetector (WASM 前端运行)
         → 检测框 + 标签 + 置信度
         → 每 2 秒发送物体清单到后端
         → 空间索引自动更新
```

### 为什么选 MediaPipe ObjectDetector
- 纯前端运行，零带宽（除了物体列表）
- WASM 加速，MacBook 上 ~30fps
- 支持 80+ 常见物体类别
- 与 FaceLandmarker 共享同一个 WASM 运行时

### 前端改动
- `useVision.ts` 并行加载 `ObjectDetector`
- 每帧同时做人脸检测 + 物体检测
- 维护 `detectedObjects: Map<string, {bbox, confidence, lastSeen}>`
- 物体出现/消失/移动时触发事件

### 后端改动
- 新增 WS 消息 `vision.objects_detected`
- 接收物体清单 → 更新空间索引
- 复合事件："检测到杯子 + 检测到键盘 → 用户在办公桌前"

### 预期效果
- "检测到人体在摄像头前" → 自动开启情绪检测
- "检测到杯子从左边移到右边" → 记录位置变化
- "检测到新物体（手机）出现" → 主动提醒"你手机放桌上了"

### 工作量：~4 小时

## Phase 3：视觉注意力机制（软件层面可模拟）
**目标**：AI 会"看"向有趣的地方，不是死盯屏幕

### 技术方案
```
物体检测结果 → 显著性评分
                - 新出现的物体：+10
                - 移动的物体：+5
                - 人脸（尤其是眼睛方向）：+8
                - 声音来源方向（如果有麦克风阵列）：+3
              → 选择显著性最高的区域作为"注视点"
              → Live2D 角色眼睛跟随注视点移动
```

### Live2D 注视映射
- 将摄像头画面坐标系（640×480）映射到 Live2D 角色 eye 参数
- `paramEyeBallX`：注视点水平位置（-1 ~ 1）
- `paramEyeBallY`：注视点垂直位置（-1 ~ 1）
- 加入随机微抖动（模拟真实人眼的扫视）

### 预期效果
- 你举起杯子喝水 → AI 眼睛看向杯子方向
- 有人从背后走过 → AI 眼睛瞟向那个方向
- 长时间没有显著变化 → AI 眼睛做随机扫视（ boredom 表现）

### 工作量：~3 小时

## Phase 4：持续视频流理解（VLM 时序分析）
**目标**：不是看单帧，而是理解"发生了什么"

### 技术方案
```
定时截图(每3秒) → 维护最近 10 帧的滑动窗口
                   → 每 15 秒把这 10 帧拼成网格图
                   → 送 VLM："描述这 15 秒内发生了什么变化"
                   → 生成事件："用户从座位站起来走向门口"
```

### 关键技术
- **帧差触发**：只有当画面变化 > 阈值时才分析（节省 Token）
- **关键帧提取**：只保留变化最大的几帧
- **时序提示词**："这是连续拍摄的 5 张照片，描述发生了什么"

### 预期效果
- "你刚才离开座位 30 秒"（检测到人物消失又出现）
- "有人在门口站了一下"（检测到新人物进入画面边缘）
- "你把咖啡杯从左边拿到了右边"

### 工作量：~6 小时

## Phase 5：深度感知 + 精确空间定位（需要硬件）
**目标**：知道"杯子在电脑左边 30cm"

### 软件方案（有限）
- **单目深度估计**：用 MiDaS 模型从前端画面估计深度图
- **物体相对位置**：基于检测框的像素坐标推算左右前后
  - 画面左侧 = 实际右侧（前置摄像头镜像）
  - 物体大小变化 = 远近变化

### 硬件方案（更好）
- **LiDAR/iPhone 深度摄像头**：直接获取点云
- **双目摄像头**：立体视觉测距
- **RGB-D 摄像头**（如 Intel RealSense）：精确到厘米级

### 预期效果
- "水杯在你左手边 20cm 的桌角"
- "你身后的书架第二层有本书歪了"
- 机器人导航："向前走 1 米，左转，伸手够桌子上的遥控器"

### 工作量
- 软件方案：~8 小时
- 硬件方案：取决于设备，+2~4 周

## 技术风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| 截图 → VLM 带宽/Token 成本高 | 每帧 0.5~2MB，VLM 按图收费 | 帧差过滤 + 压缩 + 定时触发 |
| ObjectDetector WASM 加载慢 | 首次加载 5~15MB | 懒加载，不阻塞 FaceLandmarker |
| 隐私顾虑 | 用户不想被持续录像 | 本地处理优先，可选关闭，数据不上云 |
| 前置摄像头视野窄 | 只能看到人脸附近 | 建议用户用外接广角摄像头 |
