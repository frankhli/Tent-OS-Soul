# Tent OS：AI的2D可视化成长世界 —— 完整设计文档

## 一、视觉风格定义

### 1.1 设计理念

**"温暖的几何诗"** —— 不是写实的室内渲染，而是简约、有温度、有设计感的2D插画风格。

参考风格：
- 《纪念碑谷》的简约几何 + 柔和色彩
- 《Animal Crossing》的可爱比例 + 生活气息
- 现代扁平插画（Flat Illustration）的 clean 感

### 1.2 色彩系统

### 1.3 绘制规范（Canvas 2D）

```
家具绘制原则：
- 正视图 + 轻微俯视（2.5D 感）
- 圆角矩形为主（圆角半径 = 高度的 15%）
- 阴影：offsetY=3, blur=6, color=rgba(0,0,0,0.08)
- 高光：顶部 1px 白线，模拟光照
- 描边：1.5px，颜色比填充深 20%

Avatar 在世界中的比例：
- 房间高度 400px → Avatar 高度 60px（房间的 15%）
- 家具高度 40-120px
- 走道宽度 ≥ 80px（确保 Avatar 能通过）
```

## 四、渲染架构

### 4.1 渲染层级（每帧）

```
Layer 0: 世界背景（纯色/渐变）
Layer 1: 房间地板（圆角矩形 + 填充色）
Layer 2: 房间墙壁阴影（内阴影效果）
Layer 3: 家具（按 y 坐标排序，实现遮挡关系）
Layer 4: 地毯/地面装饰
Layer 5: Avatar（在房间内的位置）
Layer 6: 特效（选中高亮、情绪粒子、交互提示）
Layer 7: UI 覆盖层（房间名称、tooltip、解锁提示）
```

### 4.2 房间渲染流程

```
function renderRoom(ctx, room, camera):
  1. 计算屏幕坐标：roomScreen = worldToScreen(room.bounds, camera)
  2. 绘制地板：
     - fillStyle = room.bgColor
     - roundRect(roomScreen.x, roomScreen.y, roomScreen.w, roomScreen.h, 16)
     - fill()
  3. 绘制墙壁阴影（内发光效果）：
     - shadowInset(roomScreen, 8, rgba(0,0,0,0.04))
  4. 绘制房间边框：
     - strokeStyle = rgba(0,0,0,0.06)
     - lineWidth = 2
     - stroke()
  5. 绘制房间名称：
     - 左上角，12px 字体，半透明
  6. 如果未解锁：
     - 覆盖层：灰色半透明 + 锁图标 + 解锁条件文字
```

### 4.3 家具渲染流程

```
function renderFurniture(ctx, furniture, roomScreenPos, camera):
  1. 计算屏幕坐标：
     fx = roomScreenPos.x + furniture.position.x * camera.zoom
     fy = roomScreenPos.y + furniture.position.y * camera.zoom
     fw = furniture.size.w * camera.zoom
     fh = furniture.size.h * camera.zoom

  2. 绘制阴影：
     - shadowOffsetY = 3 * camera.zoom
     - shadowBlur = 6 * camera.zoom
     - shadowColor = rgba(0,0,0,0.08)

  3. 根据 shape 绘制主体：
     - rounded_rect: roundRect + fill
     - rect: rect + fill
     - ellipse: ellipse + fill
     - custom: 调用专用绘制函数

  4. 绘制高光（顶部1px白线）：
     - fillStyle = rgba(255,255,255,0.2)
     - fillRect(fx, fy, fw, 1)

  5. 绘制描边：
     - strokeStyle = darken(furniture.color, 20%)
     - lineWidth = 1.5 * camera.zoom
     - stroke()

  6. 如果 hover：
     - 绘制虚线边框 + tooltip
```

### 4.4 Avatar 在世界中的渲染

Avatar 在世界中的渲染与当前 `CanvasAvatar.tsx` 基本一致，但有以下调整：
- 尺寸缩放：原尺寸 240px → 世界尺寸 60px（缩放比 0.25）
- 无背景：在世界中 Avatar 不需要背景框
- 位置：使用世界坐标，通过 camera 转换为屏幕坐标
- 阴影：底部添加小阴影增强立体感

## 五、交互机制

### 5.1 用户 → 世界

| 交互 | 触发 | 反馈 |
|-----|------|------|
| 点击空白区域 | 左键点击世界背景 | Avatar 走向该位置（如果在当前房间） |
| 点击家具 | 左键点击家具 | Avatar 走向家具 + 执行对应动作 |
| 点击 Avatar | 左键点击 Avatar | 显示状态气泡（当前情绪、系统状态） |
| 拖拽 Avatar | 按住 Avatar 拖动 | Avatar 被放置到新位置 |
| 滚轮缩放 | 鼠标滚轮 | 世界 zoom 变化（0.5x ~ 2x） |
| 拖拽平移 | 按住空白区域拖动 | 世界 camera 位置变化 |
| hover 家具 | 鼠标悬停家具 | 显示家具名称 + 功能 tooltip |
| hover 藏品 | 鼠标悬停藏品 | 显示藏品名称 + 任务来源 + 时间 |
| 双击房间 | 双击房间 | 放大到该房间（camera 聚焦） |

### 5.2 系统 → Avatar 行为映射

| 系统状态 | Avatar 目标 | 动作 | 情绪 |
|---------|------------|------|------|
| 系统告警 | 控制台 | alert | worried |
| 物理任务执行中 | 书桌/工作台 | operate | focused |
| 深度思考中 | 沙发 | think_deep | thinking |
| 监控系统 | 控制台 | monitor | calm |
| 用户检测到 | 房间入口附近 | commune | happy |
| 任务完成 | 房间中央 | celebrate | excited |
| 空闲低负载 | 沙发/地毯 | idle | calm |
| 深夜 | 沙发 | sleep | sleepy |

### 5.3 Avatar 自主行为（在世界中）

```
每 5 秒决策一次：
1. 如果有高优先级系统事件 → 走向对应家具执行动作
2. 如果物理任务执行中 → 留在工作台，随机微动作
3. 如果用户进入房间 → 走向用户方向，commune 动作
4. 如果长时间空闲 → 随机选择：
   - 在房间内随机走动
   - 走向沙发休息
   - 走向书架"看书"
   - 走向窗户"看外面"
5. 如果深夜 → 走向沙发 sleep
```

## 六、坐标系转换

### 6.1 三层坐标系

```
世界坐标系（World）
  ├── 原点：Canvas 左上角 (0, 0)
  ├── 单位：逻辑像素
  └── 范围：(0, 0) 到 (WORLD_W, WORLD_H)

房间局部坐标系（Room Local）
  ├── 原点：房间左上角 (room.bounds.x, room.bounds.y)
  ├── 单位：逻辑像素
  └── 范围：(0, 0) 到 (room.bounds.w, room.bounds.h)

屏幕坐标系（Screen）
  ├── 原点：Viewport 左上角
  ├── 单位：CSS 像素
  └── 受 camera (x, y, zoom) 影响
```

### 6.2 转换函数

```typescript
// 世界 → 屏幕
function worldToScreen(wx: number, wy: number, camera: Camera): { sx: number; sy: number } {
  return {
    sx: (wx - camera.x) * camera.zoom,
    sy: (wy - camera.y) * camera.zoom,
  };
}

// 屏幕 → 世界
function screenToWorld(sx: number, sy: number, camera: Camera): { wx: number; wy: number } {
  return {
    wx: sx / camera.zoom + camera.x,
    wy: sy / camera.zoom + camera.y,
  };
}

// 房间局部 → 世界
function roomToWorld(rx: number, ry: number, room: Room): { wx: number; wy: number } {
  return {
    wx: room.bounds.x + rx,
    wy: room.bounds.y + ry,
  };
}

// 世界 → 房间局部
function worldToRoom(wx: number, wy: number, room: Room): { rx: number; ry: number } | null {
  const rx = wx - room.bounds.x;
  const ry = wy - room.bounds.y;
  if (rx < 0 || ry < 0 || rx > room.bounds.w || ry > room.bounds.h) return null;
  return { rx, ry };
}
```
