# Tent OS：AI的2D可视化成长世界 —— 架构可行性分析

## 一、核心判断：这个构想和现有代码的契合度极高

Frank，我仔细分析了 Control UI 的全部 24 个组件 + 后端核心服务，结论是：**您的构想不是从零开始，而是在已有地基上盖楼**。四个空间系统虽然当前是独立的，但恰好可以汇聚成您描述的"三层世界"。

### 现有系统 = 构想的 70% 地基

| 您的构想层 | 已有对应组件 | 契合度 | 现状 |
|-----------|------------|--------|------|
| **世界足迹层** | `SpatialMapPanel` + `SpatialFootprintService` | 80% | 高德地图已集成，足迹/围栏/地点记忆已有，只差"主动标记里程碑" |
| **家庭认知地图层** | `VisualMemoryService` + `VisionFloatingPanel` | 60% | 已能检测物体并记录位置（"遥控器在客厅"），但位置是文本标签，未可视化到2D平面图 |
| **AI私人空间层** | `GlobalAvatarFree` + `IntentEngine` + `HomeSystem` | 70% | Avatar 已有屏幕坐标、移动意图、自主行为，但坐标系是"浏览器像素"，不是"房间坐标" |

## 二、关键洞察：Avatar 移动系统是最大优势

当前 `GlobalAvatarFree` + `IntentEngine` 已经有了一套**非常完整的2D移动系统**：
- 8 种移动意图 + 优先级打断
- ease-in-out 物理移动 + 边缘软碰撞
- 注视链（头部先转→身体跟随→最后移动）
- 到达行为（减速→环顾→转身）
- 情绪影响速度

**这意味着：Avatar 已经有"在2D空间中自主移动"的能力。**

我们只需要做一件事：**把坐标系从"浏览器像素"升级为"世界坐标系"**。

具体来说：
- 现在：Avatar 的目标位置是 `screenX = 500, screenY = 300`
- 改造后：Avatar 的目标位置是 `room = '书房', x = 120, y = 80`（相对于书房左上角的局部坐标）
- 渲染时：将房间局部坐标 → 屏幕像素坐标

这个改造的成本很低，因为 `IntentEngine` 的数学完全不变，只是加了一层坐标转换。

## 三、实施路径建议（基于现有代码的最小改动方案）

### Phase 1: "AI 的家" — 最小可行产品（2-3周）

**目标**：让 Avatar 从"在屏幕上乱逛"变成"在自己的家里有目的移动"。

**核心改动**：

1. **新建 `WorldMapPanel.tsx`**（2D Canvas 画布，与 Avatar 同技术栈）
   - 使用 Canvas 2D 绘制"房间平面图"（不是真实地理地图）
   - 默认2个房间：客厅（解锁）、书房（解锁）
   - 每个房间用简单几何图形表示：矩形房间 + 椭圆家具
   - Avatar 作为" inhabitant"在房间内显示

2. **新建 `RoomSystem.ts`**（房间坐标系管理）
   ```typescript
   interface Room {
     id: string;
     name: string;
     bounds: { x: number; y: number; w: number; h: number };
     furniture: Furniture[];
     unlocked: boolean;
     unlockCondition?: string;
   }
   ```
   - Avatar 的 `IntentEngine` 增加 `goto_room`、`goto_furniture` 意图
   - 房间局部坐标 → 屏幕坐标的映射函数

3. **改造 `GlobalAvatarFree.tsx`**
   - 从"屏幕像素漫游"切换为"房间内移动"
   - 当用户打开 `WorldMapPanel` 时，Avatar 显示在地图内
   - 当用户切换到其他面板时，Avatar 可以回到屏幕边缘（当前行为）

4. **情绪/状态的空间表达**
   - `alert` → Avatar 快速走向"控制台"家具
   - `operate` → Avatar 站在"工作台"前做操作动画
   - `think_deep` → Avatar 走向"沙发"坐下思考
   - `monitor` → Avatar 站在"显示屏"前扫视

**Phase 1 交付物**：用户打开 WorldMapPanel，看到 AI 在自己的2D家里走动、工作、休息。

### Phase 2: "虚实映射 + 成长解锁"（2-3周）

**目标**：摄像头看到的东西出现在 AI 的家里；完成任务解锁新房间。

**核心改动**：

1. **视觉记忆 → 家具映射**
   - `VisualMemoryService` 检测到的物体（沙发、冰箱、书桌）→ 在 `WorldMapPanel` 的对应房间显示为家具图标
   - 物体位置从文本标签（"在客厅"）→ 2D 坐标（客厅内的 x/y）
   - AI 会说："我看到客厅沙发上有个遥控器" → 地图上显示遥控器图标在沙发家具上

2. **任务成果 → 智慧藏品**
   - 每个完成的任务 → 生成一个"智慧藏品"（PPT 变书架上的书、Excel 变数据水晶）
   - 藏品自动摆放在对应房间：代码任务→工坊、写作任务→书房

3. **房间解锁系统**
   - 新房间预设解锁条件：
     - 工坊：完成 10 个代码任务
     - 图书馆：完成 20 个知识类任务
     - 温室：完成 5 个创意任务
   - 解锁时播放动画 + Avatar 兴奋地跑向新房间

4. **世界足迹与家的联动**
   - `SpatialMapPanel` 的足迹 → 在 `WorldMapPanel` 的"世界墙"上显示为旅行照片
   - 去过的地方越多，"世界墙"越丰富

**Phase 2 交付物**：AI 的家会随任务完成而丰富，摄像头看到的物品会出现在地图上。

### Phase 3: "深度互动 + 记忆空间化"（2-3周）

**目标**：家和 AI 一同成长，记忆有空间位置。

**核心改动**：

1. **记忆空间化**
   - `MemoryPanel` 的记忆条目 → 在 `WorldMapPanel` 的"记忆墙"上显示为便签/照片
   - 重要记忆可以"钉"在特定房间（和用户的第一次对话 → 钉在客厅）

2. **AI 的私人空间**
   - "梦境空间"：REM 阶段的记忆整理 → 在地图上显示为梦境气泡
   - "情绪角落"：AI 难过时会去的私密空间

3. **用户互动**
   - 点击家具 → 查看该家具相关的任务/记忆
   - 拖拽 Avatar 到新房间 → Avatar 会记住这个偏好
   - 点击藏品 → 查看任务详情和回顾

**Phase 3 交付物**：家和 AI 的记忆、情绪、成长深度绑定，成为真正的"数字生命栖息地"。

| 之前的担心 | 2D世界如何解决 |
